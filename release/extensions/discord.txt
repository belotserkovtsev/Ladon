# ladon extension: Discord
#
# Подключается через config.yaml:
#   allow_extensions: [discord]
#
# Discord ecosystem: приложение, gateway, CDN, голос/видео (media),
# party-games, активити, мерч, статус. eTLD+1 раскрытие в dnsmasq
# покрывает поддомены автоматически (gateway.discord.gg,
# cdn.discordapp.com, edge.discord.gg и т.д.).
#
# Голос/видео — отдельная история: Discord-клиент после WSS-handshake
# с gateway.discord.gg получает IP+port voice-сервера и шлёт UDP RTP/STUN
# напрямую, без DNS-резолва конкретного хоста. Один discord.media в
# списке доменов накачивает в ladon_manual только те /24, которые
# успели зарезолвиться где-то ещё (probes, прошлые сессии). Дыры в
# voice /19 проявляются как «voice connected, но всех не слышно» →
# 5000ms ping → разрыв. CIDR-блок ниже гарантирует покрытие всего
# голосового пула независимо от DNS.
#
# Источники CIDR:
#   AS13335 (Cloudflare) — два пула для Discord voice:
#     - 162.159.128.0/19 (Magic Transit egress, основной с 2022)
#     - 104.16.0.0/12   (anycast, региональный failover; разные /16
#                        по регионам — Финляндия зачастую 104.25.x,
#                        наблюдали 104.29.x в EU. Узкий /16 поломает
#                        voice пользователям в других регионах.)
#   AS49063 (Discord, Inc.) — собственное IP-пространство (API,
#     image CDN, RTP legacy). 14 диапазонов из ASN-аллокации.
